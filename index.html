Bienvenue sur le site du Memory de l'ENI.
Venez jouer avec nous et faites le meilleur score possible!

Règles du jeu:

    Retourner deux cartes:
  • Si elles sont identiques: le jeu laisse les cartes visibles.
  • Si elles sont différentes : les cartes sont retournées faces cachées là où elles étaient.
  • La partie est terminée lorsque toutes les paires sont assemblées
Si vous voulez vous souvenir de votre score :
inscrivez vous via le formulaire dédié.
Le jeu garde vos 5 meilleurs scores par grille en mémoire.
Pour une meilleure expérience du jeu,
tournez votre téléphone en mode paysage

Projet : HTML - CSS - JS

Jeu du Memory

1. Objectifs:

  • Mise en application de la technologie web enseignée :
    • Dans le cadre du cursus DW2M, ce projet arrive après les modules HTML, CSS et JavaScript, c'est à dire un mois de formation.
  • Réaliser un premier projet. Durée donnée : 35h.
  • Mettre en oeuvre les connaissances en HTML, CSS et JavaScript basique avec manipulation du DOM et vérification de formulaire.
  • Commencer aussi à apprendre GIT de façon basique.
    • Créer un compte sur Gitlab ou Github pour pouvoir ensuite partager le projet à un possible recruteur (à définir avec le formateur).
    • Ici l’utilisation de GIT est basique. Juste travailler sur une branche et sauvegarder le travail, pour avoir un historique et savoir les commandes basiques : git add commit push, pull, ainsi que de savoir-faire un fichier .gitignore.
  • Chiffrer les tâches et commencer à gérer un projet.
  • À la fin du projet, restituer au formateur le zip . Remplir le document d’auto-évaluation.

  • Document de vision : présente le projet, ses objectifs, ainsi qu’une vision globale des fonctionnalités attendues (Voir ci dessous).

1. Présentation du projet:

La société ENI souhaite développer pour ses stagiaires une plateforme web leur permettant de jouer à un jeu de memory, de pouvoir sélectionner le type de plateau, la dimension et de s’inscrire puis visualiser leur score et les meilleurs scores.

2. Positionnement:

Nos Forces

La présence d’un grand nombre d’apprenants répartis sur différents sites en France.
Les opportunités

L’accessibilité au web aisée en raison de l’accès aux ordinateurs de bureau et l’utilisation courante des smartphones, tablettes doit permettre de faire vivre la plateforme.
La spécialisation informatique de l’ENI doit permettre la réalisation de la plateforme.
Nos ambitions

Développer une application responsive et soignée permettant aux apprenants de jouer à des parties de memory, de choisir leur plateau, de s’inscrire et de visualiser leurs meilleurs scores.
Les résultats attendus

Permettre de s’amuser et de sauvegarder son meilleur score.

2.1 Description du problème:

Mettre en place un jeu de Memory permettant la sélection de différents plateaux comme par exemple la possibilité de jouer avec les lettres du scrabble ou de jouer avec des animaux, des légumes et par la suite de pouvoir rajouter facilement son environnement de jeu.

  • Sauvegarder dans le localStorage du navigateur les différents scores et garder les 10 meilleurs affichés.
  • Sauvegarder l’inscription de l’utilisateur avec une sauvegarde du score et de ses dernières parties dans le localStorage.
  • Mettre en place un formulaire pour l’inscription de l’utilisateur et ensuite pouvoir sauvegarder ses préférences en matière de memory et ses derniers scores.

... etc. On nous fourni des captures d'écrans des différentes pages et fonctionnalités attendues
